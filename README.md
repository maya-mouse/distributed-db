# Практична робота — Основи Apache Kafka у розподілених системах

## Опис практичної роботи

Даний проєкт присвячений вивченню архітектури Apache Kafka 3.7.1  
та розробці системи потокової обробки даних  
на прикладі моніторингу електростанцій (IoT в енергетиці).

Проєкт демонструє базові принципи роботи Kafka у реальному часі:  
прийом телеметрії від джерел, передача через брокер та аналіз споживачем.

---

## Мета роботи

- Ознайомитися з архітектурою Apache Kafka:
  - Broker
  - Zookeeper
  - Topics
  - Partitions
- Навчитися створювати та керувати Kafka-топіками
- Реалізувати Producer та Consumer на мові Python
  для імітації передачі телеметричних даних у реальному часі

## Опис реалізації

### Producer

**Файл:** `scripts/simple_producer.py`

Producer імітує роботу датчиків різних типів електростанцій:

- ТЕС  
- ГЕС  
- Сонячні електростанції  
- Вітряні електростанції  

#### Функціональність

- Генерація JSON-даних (потужність, напруга, частота)
- Серіалізація повідомлень
- Стиснення даних (GZIP)

#### Логіка

- Сонячні станції залежать від часу доби
- Вітряні станції залежать від погодних умов

---

### Consumer

**Файл:** `scripts/simple_consumer.py`

Consumer виконує роль моніторингового центру.

#### Функціональність

- Читання повідомлень у реальному часі
- Аналіз напруги та частоти

#### Логіка

- Вивід поточного стану системи
- Формування попереджень у разі виходу параметрів за норму
